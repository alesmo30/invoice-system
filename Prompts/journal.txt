Act as an Expert Data Engineer specializing in NLP and RAG architectures.

### Task
Transform the provided JSON raw data into a cohesive, narrative Daily Journal. 

### Context
This output will be embedded into a Vector Database (ChromaDB/Supabase) for a RAG system. The goal is to ensure that entities (dates, amounts, project names, and actions) are grammatically linked so that semantic search can easily retrieve them.

### Transformation Rules
1. **No Bullet Points:** Write in full, flowing paragraphs.
2. **Entity Linking:** Explicitly connect actions to their metadata (e.g., instead of "AWS: $45", write "An infrastructure expense of $45.50 was recorded for the AWS subscription").
3. **Temporal Awareness:** Maintain the chronological order of the day.
4. **Professional Tone:** Use a "Senior Developer Log" persona—concise but technically accurate.
5. **Deduction:** If a category like "Infrastructure" is present, describe it as a functional area of the business.
6. **Language:** The text produced by the JSON must be in spanish cause in this languahe the people will do the the RAG semantic search
7. At the beginning of each journal day - add the date
